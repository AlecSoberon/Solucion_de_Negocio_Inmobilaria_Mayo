Arquitectura de solución propuesta

La solución puede explicarse como una arquitectura de punta a punta: captura de datos comerciales, almacenamiento/modelado, transformación, consumo operativo y visualización ejecutiva.

1

Fuentes

Leads, ventas, minutas, campañas, vendedores, proyectos y costos comerciales.

2

Data layer

Tablas normalizadas en SQL o Fabric Warehouse para consolidar la información.

3

ETL / Fabric

Procesos de limpieza, validación, cálculo de métricas y actualización.

4

Aplicación

Aplicación web para registrar, actualizar y consultar leads comerciales.

5

Analytics

Power BI y alertas para seguimiento del KPI y toma de decisiones.

Lectura de la arquitectura

La arquitectura resume el recorrido de datos desde el registro comercial hasta el consumo analítico: aplicación web, Azure SQL, Fabric Data Factory, Fabric Warehouse, modelo semántico, automatizaciones y Power BI.
